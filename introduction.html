경험들

~2020

인덕대학교 정보통신공학과에 16학번으로 입학해 2018년에 해군 복무 후 19년도 말 제대했습니다. 프로그래밍 과목에선 모두 A 를 맞고 다른 회로과목이나 전기통신 과목은 F 를 맞았언 기억이 납니다. 학과가 너무 포괄적이여서 전과를 고려했었습니다.

2021

복학 후 공개SW 에서 주최하는 Open Source Software 개발자 대회 예선에 빅데이터/AI 분야로 참가했었습니다. 유저가 자신의 옷을 찍어서 올리면 그에 어울리는 상하의(하상의)를 추천해주는 APP 을 개발해보려고 했습니다. 쇼핑몰 피팅 사진을 Numpy 이미지배열을 통해 데이터셋으로 만들고, tensorflow 와 opencv 라이브러리를 통해 다중선형회귀분석으로 옷의 RGB값을 머신러닝시킨 후 출품했습니다. 직접 데이터전처리를 했었기때문에 근본적인 데이터 양이 부족하여 결과가 만족스럽게 나오진 않았던 것이 아쉬웠습니다.

2022

단기계약으로 사무직근무할 때 반복되는 업무를 줄이는 python 코드를 작성하여 업무 시간을 90% 줄인 경험이 있습니다. 22년도 1월~2월 두달동안 ABLE C&C 에서 단기 계약직으로 근무할 때, 기존 다른 회사들과의 계약을 갱신 혹은 해지 하는 반복 업무를 맡았었습니다. 3가지 다른 DB에서 추출한 엑셀 파일을 하나로 합치는 python 코드를 작성하여 기존 수작업으로 20~30분정도 걸리는 업무를 1분~2분으로 단축시켜 근무를 여유롭게 할 수 있었습니다.

7만 조회수의 웹게임을 개발한 경험이 있습니다. 로스트아크라는 게임 속 미니게임을 연습할 수 있는 웹게임을 Unity 를 통해 개발하여 커뮤니티에 업로드해 7만조회수를 기록하고 유명 스트리머 방송에서도 다뤄졋었습니다.

팬페이지 개발을 위해 CI/CD Pipeline 을 구축했었습니다. blackpink 팬페이지 개발을 위해 젠킨스 컨테이너에 깃허브 리포지토리 웹훅을 트리거하게 한 후, 빌드된 이미지를 hub.docker 로 푸시해서 앤서블을 통해 Prod 환경으로 배포하게했었습니다. 제작중 프론트앤드의 퀄리티가 너무 떨어져 상업성이 없다고 생각해서 개발을 중단했습니다.

EKS 로 관리되는 인프라의 설계 및 구축 프로젝트를 진행했었습니다. 구름 쿠버네티스 전문가 양성 과정(국비지원)의 마지막 프로젝트에서 k8s 로 관리되는 인프라의 전반적인 설계에 참여했었고, 구축 부분에서는 CI/CD 파트를 주로 맡았습니다. Jenkins+ArgoCD+ECR+EKS 의 구성을 직접 설계 및 구축했고, jenkins build server 는 클러스터 밖에, argocd 는 클러스터 내의 별도 namespace 에 설치했다. 앱은 로드밸런서 타입으로 서비스되도록 하였다.

2023

가상화폐 자동매매 워크로드의 변경으로 인해 증가한 인스턴스 유지비용을 서버리스 아키텍쳐로의 마이그레이션을 통해 7배 감소시켰습니다. FB의 Prophet 라이브러리를 사용한 인공지능 종가 예측 워크로드를 추가하려면 4GB 이상 RAM 이 필요하고, 이에 t2.medium이상온디멘드 약 $34의 인스턴스 유형이 필요했었습니다. 이에 해당 워크로드를 Lambda + DynamoDB + CloudWatch 로 마이그레이션하고 자동으로 실행되게 하여 결과적으로 $6 이내의 월 요금이 나오도록 구축했습니다.

2023~

앞으로 가상화폐 자동매매 워크로드를 IaC 로 관리해볼 예정입니다. 애초에 AWS-CDK python 으로 관리해보려고 했으나 CDK container 와 Jenkins 의 통합 과정에서 오류가 많아 뒷순위로 미뤘던 작업이여서 추후에 적용해보려고 합니다.

인디게임을 개발해서 구글 마켓에 출시해보려고 합니다. 간단한 인디게임을 chatGPT 와 생성AI를 사용해서 Unity 로 제작하고, websocket API 혹은 Data Stream 을 통해 서버 또한 구축해서 배포해보려고 합니다.

29살 혹은 30살이 되면 데이터 혹은 AI 관련 학부 대학원으로 진학해서 석사학위를 위해 공부하고 싶습니다. AIOps 가 저의 최종 목표이며 스카이넷처럼 거대한 네트워크를 설계 해보고싶습니다.

성격

18살때부터 성격 유형 검사시 ENTJ 혹은 INTJ 유형이 나왔었고 최근까지도 ENTJ 유형이 나오고 있습니다. 저는 자기 객관화가 잘 되어있습니다. 1년이 끝나면 그 해에 대한 보고서를 작성해서 잘해온것과 부족한것을 파악하고 발전을 위해 스스로를 연구하고있습니다. 어려운 문제를 시도하거나 새로운 기술에 도전하는 것을 좋아하고 여러 정보를 합성하거나 분석하고 통찰하는 것을 즐깁니다. 계획을 세우고 실천에 옮기는 타입입니다. 실제로 연-분기-월-주-일 별 계획을 세우고 체크리스트로 실천해나가고 있습니다. 사람을 사귀는데에도 큰 어려움은 없어서 제 배경지식이 닿는 한 업무적 소통에는 문제가 없었고 스몰토크도 공유할 수 있습니다.

개인적으로 효율을 중시하고 논리에 의한 합리적 선택을 우선시하기 때문에 종종 직설적일때가 있습니다. 때문에 상당히 비효율적인 직무가 많았던 군대에서 적응하는데 상당히 힘들었던 기억이 있습니다. 당시 군생활 상담을 통해 유화적인 태도를 갖는것이 하나의 가면을 쓰는 것이고 이는 사회생활에서 상당히 좋은 카드가 될 수 있다는 것을 배웠었습니다. 업무에서의 소통시 상대방의 감정을 잘 신경쓰지 못합니다. 업무에 있어서 사람들의 감수성 혹은 감정적인 의견은 대체로 묵살하는 편입니다. 이는 공사를 구분하는 편인데, 사석에서 상대방의 기분을 잘 이해하지 못하고 너무 이성적이라는 얘기를 어렸을때부터 들어와서 최대한 배려하려고 노력하고 있습니다. 하지만 업무에 있어서 팀원이 비효율적으로 일할때 피드백을 직설적으로 하는 편이며, 소통시 상대방이 지나치게 감정적인 의견을 내새운다면 공감하기 힘듭니다.
